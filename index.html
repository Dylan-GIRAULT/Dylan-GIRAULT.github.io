The game created in C89 language : Blocus

The game was created during the first semester of my first year of DUT in IT.

Game instuctions

After choosing a square grid whose size is between 3 and 9, one after the other, the two players
place their pawn and then condemn a square. Then, each player can move his pawn to an adjacent
square (including diagonally) and again condemn a square. The first player who cannot move his pawn will have lost.

Although this was not requested, my comrade and I made graphic additions such as the coloring of the box
where the mouse is located, but also the differentiation of the boxes condemned to know which player has blocked which box.





In addition, to avoid having a BOT that is too random, we have introduced the obligation that the box condemned by the BOT is around the player, which makes the game more complex.













Cooperating with a comrade was the greatest difficulty, which means having a code that is readable and easy to understand. This problem solved, the exercise becomes easier than it seems and above all, enabled me to experience new things learned during the lessons.

The game created in Java langage : Candycrush Lite

The game was created during the second semester of my first year of DUT in IT.

Game instuctions

In this game, a grid is filled with blocks of three types and the goal is to clear it. A group of
adjacent blocks of the same type can be removed by clicking on them. The remaining blocks are rearranged
in order to fill the holes and we start again until the grid has been emptied or only isolated blocks are left.



Opening the game from a file

The game is able to retrieve a file respecting a certain syntax in order to create the game board.
You can see the content of the "test.txt" file opposite, used as an example in the video.







Allowing files to be imported increases the risk of bugs, especially if the file does not comply
with the desired syntax. For this reason, the program is able to handle errors and report them to the user.

Error message of a file not found




Error message of an unacceptable character (different from "R", "V", "B")




Error message of an incorrect size of the array




This project, because of the use of the java language, requires a much more extensive organization.
This forces us to think about how to make our code accessible to his teammates, without causing disagreement between the created class and the user of the code who must be able to use it without having to know its structure.
